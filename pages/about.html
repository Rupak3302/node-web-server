About this project

How the
Server Works

This server is built using only Node.js's built-in http, fs, and path modules. There are no external packages. The server listens on port 3000, reads the request URL, matches it against a route map, and asynchronously reads the appropriate HTML file from disk.

server.js — core logic 
// Route map: pathname → { file, status }
const routes = {
  '/home':    { file: 'home.html',    code: 200 },
  '/about':   { file: 'about.html',   code: 200 },
  '/contact': { file: 'contact.html', code: 200 },
};

// Async helper — non-blocking file read
function servePage(res, file, code) {
  fs.readFile(path.join(PAGES_DIR, file), 'utf8', (err, data) => {
    if (err) { /* 500 fallback */ return; }
    res.writeHead(code, { 'Content-Type': 'text/html' });
    res.end(data);
  });
}

Request Lifecycle

Step 01
Browser makes an HTTP GET request
e.g. GET /about HTTP/1.1 arrives at port 3000.
Step 02
Server parses the URL pathname
The URL API strips query strings cleanly.
Step 03
Route lookup
Pathname is matched against the route map. Unknown paths fall through to the 404 handler.
Step 04
Async file read
fs.readFile reads the HTML from disk without blocking the event loop.
Step 05
Response sent
Correct HTTP status code + Content-Type: text/html header + HTML body.
